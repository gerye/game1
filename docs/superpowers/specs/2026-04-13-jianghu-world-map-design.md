# 江湖世界地图系统设计文档

**日期**：2026-04-13  
**状态**：待实现  

---

## 一、设计目标

当前游戏缺乏"底层世界"——角色只在大事件中出现，门派是纯粹的身份标签。本系统引入一张持续运行的六边形世界地图，角色平时在地图上自由活动，门派以城池控制的形式真实存在并发展。玩家作为超脱门派的**仲裁者**，可随时打开游戏查看格局演化，偶尔插入特殊事件或处理角色命运节点，无需频繁介入。

---

## 二、世界地图结构

### 2.1 地图形状与地形

地图为**不规则六边形网格**（约 40×40 格半径），沿用现有战斗系统的六边形技术，但尺度远大于战斗场。

边界地形（不可进入/特殊）：
- **北方**：草原（边疆，稀疏城池）
- **南方 & 西方**：大山（天险，阻断连通）
- **东方**：大海（海岸线，仙岛临海）

中原腹地地形（可通行，影响产出和战斗加成）：平原、森林、水域、岩石、熔岩——沿用现有派系地形亲和体系。

### 2.2 六派初始位置

| 门派 | 位置 |
|------|------|
| 青云门 | 中北 |
| 少林 | 中南 |
| 魔教 | 西北 |
| 教廷 | 正西 |
| 仙岛 | 东偏北（临海） |
| 魂殿 | 西偏南 |

### 2.3 城池层级

每个门派拥有：
- **总部**（1座）：特殊城池，永久属于本门派，不可被攻占，是门派存亡的最后防线
- **大城**（3座）：位于总部周边，门派核心领土
- **小城**（散布）：位于门派交界处的外围地带，是主要争夺对象

城池控制周围若干格子，控制范围染上门派颜色，形成势力版图。未被任何门派控制的格子为**中立地带**（灰色）。

## 三、资源系统

### 3.1 声望值（Prestige）

- **来源**：江湖大事件（见第五节）
- **用途**：唯一用于城池扩张的货币，**不持续消耗，一次性花费**
- **不可用于**：角色培育

**城池操作费用（基准单位 T = 1次江湖争霸冠军声望）**：

| 操作 | 声望费用 |
|------|----------|
| 占领空白小城 | 500 |
| 攻打敌方小城（攻城战）| 1000 |
| 攻打敌方大城（攻城战）| 2500 |

### 3.2 金币（Gold）

- **来源**：控制城池持续产出（每「时节」结算一次）
- **产出量**：由城池规模等级 + 周围敌对城池数量决定（边境城池风险更高但产出更高）
- **用途**：角色培育消费
  - **门派秘境**：花费金币送单个角色进入秘境闯关
  - **拍卖会**：不定期出售稀有物品（S级血脉、装备），门派竞价购入
- **当前状态**：拍卖会接口预留，金币暂为纯数字显示，消费场景后续实现

## 四、角色在地图上的生活

### 4.1 角色状态

每个角色平时处于三种状态之一，**完全由门派 AI 管理**，玩家无需介入：

| 状态 | 说明 | 效果 |
|------|------|------|
| **驻守** | 留在城池内 | 为城池贡献控制力，缓慢积累经验值 |
| **漫游** | 在地图格子间移动 | 触发地图奇遇事件，是日常声望/经验/金币零散来源 |
| **出征** | 参与大事件 | 临时离开驻守位置，参加混战/竞标赛/排位赛 |

### 4.2 地图奇遇事件（小事件）

游戏每经过一个「时节」，每个漫游中的角色触发一次事件，类型根据所在地形和周边势力格局随机生成。事件效果包括：经验获取、金币小额产出、声望小额变动、触发秘境入场机会等。

这一机制沿用并扩展现有的战前随机事件系统（`prebattle.js`）。

### 4.3 边界冲突与重伤状态

当两个**不同门派**的漫游角色在边界格子相遇时，概率触发**单挑**：
- 败者进入**重伤状态**：需回到己方城池休养，下一场正式战斗属性受到一定惩罚
- 胜者获得小额声望加成
- 重伤状态在城池驻守若干时节后自动解除

## 五、大事件系统

### 5.1 事件类型与触发条件

| 事件 | 触发条件 | 参与范围 |
|------|----------|----------|
| **江湖争霸** | 所有角色完成一轮地图奇遇后自动触发 | 全门派 |
| **武道会** | 定期自动触发（条件与现有逻辑一致） | 全门派 |
| **排位赛** | 定期自动触发 | 全门派 |
| **攻城战** | 门派 AI 主动发起（消耗声望）| 攻守双方 |
| **全员决战** | 某门派三座大城全部被占时自动触发 | 相关门派 |
| **拍卖会** | 仲裁者手动触发（接口预留）| 全门派（竞价） |

玩家可随时**手动插入**江湖争霸、武道会或拍卖会，作为仲裁者干预格局。

### 5.2 声望奖励分配（具体数值）

江湖争霸：只有获胜门派获得 **100声望**，其余门派无奖励。

武道会：

| 名次 | 声望 |
|------|------|
| 第一 | 800 |
| 第二 | 480 |
| 前四（3-4名） | 250 各 |
| 前八（5-8名） | 100 各 |

排位赛（各档略高于武道会，多一档前十二）：

| 名次 | 声望 |
|------|------|
| 第一 | 1000 |
| 第二 | 600 |
| 前四（3-4名） | 300 各 |
| 前八（5-8名） | 150 各 |
| 前十二（9-12名）| 50 各 |

## 六、门派消亡与吞并

当某门派的**三座大城全部被其他门派占据**时，自动触发**全员决战**：

- 该门派全体弟子 vs 所有占据了其大城的门派联军
- **原门派获胜**：夺回三座大城，恢复正常运转
- **攻方获胜**：获胜门派控制总部及三座大城；被灭门派全体弟子并入获胜门派，继续以新门派身份参与游戏

随着吞并不断发生，强门派弟子数量增多、实力雄厚，最终走向**一统江湖**。

---

## 七、玩家（仲裁者）干预点

玩家**不需要**管理角色日常行为，只需在以下场景介入：

1. **插入特殊事件**：觉得格局太稳，手动触发武道会或拍卖会打破平衡
2. **角色命运节点**：角色触发血脉机会、技能升级选择等成长岔路，排队等待处理
3. **观察格局**：随时打开地图查看六派版图和三维数值（实力/声望/金币），无需任何操作

## 八、技术实现要点

| 模块 | 说明 |
|------|------|
| `src/world-map.js` | 地图数据（格子、地形、城池、控制权）+ 渲染 |
| `src/faction-state.js` | 门派三维状态（实力/声望/金币）+ 城池控制逻辑 |
| `src/world-tick.js` | 时节推进引擎：角色状态更新、小事件触发、资源结算 |
| `src/world-events.js` | 大事件调度：触发条件检测、声望分配、攻城/决战逻辑 |
| `src/world-ui.js` | 世界地图 UI：Canvas 渲染、门派颜色填色、仲裁者操作面板 |
| 现有模块改造 | `prebattle.js` 扩展为地图小事件系统；大事件模块对接声望分配接口 |

地图渲染沿用现有 Canvas 六边形体系，坐标系和格子逻辑可直接复用 `battle-system.js` 的底层数学。
