# 游戏系统说明 1.1.0

## 1. 角色生成

图片只负责生成：

- 唯一编码
- 头像
- 底色信息

潜力等级不再由图片自动直接判定，而是由人工录入三项：

- 颜色数
- 复杂度
- 对称度

潜力映射表定义在：

- `src/rule-tables.js`

## 2. 属性系统

一级属性：

- 力量
- 体质
- 敏捷
- 智力
- 精神

二级属性由一级属性推导：

- 物攻 = 力量 × 3
- 法攻 = 智力 × 3
- 物防 = 体质
- 法防 = 精神
- HP = 100 + 体质 × 10

角色详情中的属性使用：

- 白字 + 绿字 = 最终值

## 3. 技能系统

主动技能只受 CD 限制，不再使用 MP。

主动技能基础 CD：

- `E/D/C/B/A/S/SS/SSS = 20/19/18/17/16/14/12/10 秒`

状态攻击类：

- 灼烧攻击
- 麻痹攻击
- 睡眠攻击

规则：

- 技能本体伤害固定为 1 倍普通攻击
- 额外附加伤害随等级成长：
  - `E = 10%`
  - `D = 20%`
  - `C = 30%`
  - `B = 40%`
  - `A = 50%`
  - `S = 60%`
  - `SS = 70%`
  - `SSS = 80%`

## 4. CD 与状态时长

### CD 倍率

由：

- `(智力 + 精神) / 2`

决定。

### 状态倍率

由：

- `(力量 + 体质) / 2`

决定。

两者都使用同一条公式：

```text
倍率 = (x + 300) / (3x + 300)
```

因此：

- `x = 300` 时，倍率为 `50%`
- `x -> 无穷` 时，倍率趋近 `33%`

## 5. 状态系统

状态统一定义在：

- `src/status-effects.js`

典型状态：

- 束缚
- 灼烧
- 麻痹
- 睡眠
- 死里逃生
- 吃坏肚子
- 荣誉状态
- 血脉附体状态

当前灼烧：

- 持续 `2.5 秒`
- 每秒损失最大生命值 `2.5%`
- 每 `0.1 秒` 结算一次

## 6. 荣誉系统

当前主要荣誉层数：

- 百战百胜：每层 `+0.5%`
- 江湖大派：每层 `+2%`
- 天之骄子：每层 `+5%`
- 稍逊风骚：每层 `+2%`
- 后起之秀：每层 `+1%`

## 7. 血脉系统

血脉统一定义在：

- `src/bloodlines.js`

血脉附带：

- 常驻状态
- 逆天改命次数
- 赠送技能
- 名字前缀符号

SSS 血脉支持任务获取，任务规则在：

- `src/bloodline-tasks.js`

## 8. 战斗系统

战斗核心位于：

- `src/battle-system.js`

地图：

- 六边形格子
- 单位按格占位
- 不可穿墙
- 单位不可重叠

AI 状态：

- `seek`
- `encounter`
- `battle`

超时保底：

- 战斗满 5 分钟自动判定
- 江湖争霸：
  - 先比存活人数
  - 再比总剩余血量百分比
  - 再比总剩余血量绝对值
- 武道会 / 排位赛：
  - 先比剩余血量百分比
  - 再比剩余血量绝对值

## 9. 排位赛

排位赛使用：

- 瑞士轮
- 前 12 进入淘汰赛

排位相关模块：

- `src/ranking-utils.js`
- `src/ranking-ui.js`
- `src/ranking-history.js`

每次排位会保存：

- 每轮配对表
- 瑞士轮最终成绩
- 淘汰赛树
- 历届快照

## 10. 快速推演与无尽推演

快速推演：

- 自动推进争霸 / 秘境 / 武道会 / 排位赛

无尽推演：

- 先完成快速推演前段
- 然后持续循环

规则在：

- `src/fast-sim.js`

## 11. 历史记录

江湖大事记：

- 保留关键里程碑
- 血脉获得、排位前八、武道会等条目优先保存

个人小传：

- 记录个人里程碑
- 排位名次、血脉获得、武道会高名次优先保存
